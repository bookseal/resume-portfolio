# 포지셔닝 전략: Applied AI Engineer - Agent GYM (Upstage 194880)

> 이 문서는 `Upstage_JD_194880.md` 직무에 맞춘 개인화 지원 전략입니다.

## 1. JD 핵심 요구사항 및 분석
이 직무는 연구 성과를 실제 프로덕션 수준의 안정적인 AI 시스템으로 전환하는 **응용(Applied) AI 엔지니어**를 찾습니다.
- **핵심 키워드:** LLM/Agent, Tool Calling, 외부 시스템 연동, 프레임워크(FastAPI, LangChain), 관측 가능성(Observability), 연구→제품 전환 실행력

## 2. 포지셔닝 (한 줄 요약)
**"최신 LLM 연구(LangGraph, Solar)를 장난감 수준을 넘어, Tool Calling과 외부 API 연동이 포함된 실제 프로덕션(Production) 에이전트로 배포해 내는 실행 엔진"**

## 4. 이력서/자소서 작성 방향
- **경력 기술:** 에이전트 툴(Claude Code, NotebookLM 등)을 현업 워크플로우에 직접 도입해 생산성을 올린 경험과, 그 과정에서의 프롬프트/시스템 설계 역량을 강조.
- **프로젝트 강조:** BookToss를 압도적인 메인 프로젝트로 끌어올림. LangGraph 상태 전이, 장애 복구, Tool Calling 등 Agent 엔지니어링의 딥한 부분을 기술적으로 증명.
- **연차 방어:** "에이전트 분야는 누구나 1~2년 차입니다. 연차가 아닌 최신 기술의 찐 적용(Applied) 경험이 중요합니다"라는 프레임으로 주니어 연차를 방어하고 최신 포트폴리오를 앞세움.
