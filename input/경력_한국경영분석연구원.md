# 경력 — 한국경영분석연구원 (Korea Institute of Business Analysis & Development)

**기간**: 2026.05 ~ 현재
**역할**: Lead AI FDE (Front-end & Full-stack Development Engineer)

3인 규모의 팀을 리드하며, 개발의 방향을 조정하고 온보딩을 수행하고 있습니다. 기술적 의사결정, AI 기반 업무 자동화, 프로젝트 관리 및 리더십 측면의 주요 성과는 다음과 같습니다.

### 1. 프론트엔드 아키텍처 설계 및 레거시 시스템 모던화 (Front-End & Architecture)
*   **엑셀/VBA 기반 수작업의 웹 서비스(DB) 전환 리드:** 파편화되어 있던 사내 직원 정보, 학력, 자격증, 업무 분류 데이터를 통합하여 SQLite 기반의 데이터베이스를 구축하고 Streamlit을 활용한 사내 맞춤형 웹 대시보드로 성공적으로 전환했습니다.
*   **비즈니스 목적에 맞춘 UI/UX 기술 스택 의사결정:** 장기적인 유지보수성과 B2B 사내 시스템의 특성을 고려하여, 오버엔지니어링(React 도입 및 모바일 반응형 웹 구현)을 방지하고 코드 구조가 단순한 Streamlit을 유지하도록 기술적 의사결정을 주도했습니다.
*   **임원진(최종 사용자) 맞춤형 인쇄용 UI 고도화:** 스마트폰이나 모니터보다 종이 인쇄물을 선호하는 경영진의 업무 스타일을 정확히 파악하여, 웹상에서 복잡한 자격증-업무분류 매핑 데이터를 A3/A4 인쇄 포맷에 최적화된 CSV 및 Excel 파일로 한 번에 다운로드할 수 있는 기능을 기획하고 구현했습니다.
*   **직관적인 데이터 시각화 및 대시보드 구현:** 엑셀로 관리되던 조직도를 시스템 내에서 실시간으로 연동되도록 동적 UI를 구현하고, 데이터 수정이 불가능한 파생 데이터(Read-Only)는 회색 배경으로 분리하여 사용자의 입력 실수를 방지하는 UX 개선을 기획했습니다. 또한, 자격증 만료 30일 전 경고 알림(빨간색/노란색 등 시각적 강조) 기능을 기획하여 도입했습니다.

### 2. AI 에이전트 기반 개발 파이프라인 및 워크플로우 구축 (AI & Automation)
*   **LLM 기반 협업 자동화 시스템 구축:** 개발자 간의 커뮤니케이션과 코드 리뷰를 효율화하기 위해 Claude Code, Codex 등 AI 에이전트를 도입하여 Github Issue 자동 생성, Pull Request(PR) 작성, 코드 리뷰 및 코멘트 작성을 자동화하는 혁신적인 파이프라인을 구축했습니다.
*   **회의록 자동화 및 지식 자산화 (RAG 도입 기반 마련):** Microsoft Teams의 실시간 자막(STT)을 캡처하는 오픈소스 확장 프로그램(Teams Caption Saver)을 도입하여 회의 내용을 텍스트화하고, 이를 NotebookLM과 연동하여 회의 요약, 업무 히스토리 검색, 문서화를 100% 자동화했습니다.
*   **비전 API를 활용한 보안 처리:** 화면 캡처 이미지 내의 민감정보나 개인정보를 AI(Claude)가 자동으로 인식하고 모자이크(Redaction) 처리해 주는 기능을 업무 워크플로우에 테스트 및 적용했습니다.
*   **오픈소스 기반 피드백 포털 도입 기획:** HTML/React로 대시보드를 처음부터 하드코딩하는 대신, 사내 요구사항을 관리하고 피드백을 수집할 수 있는 검증된 오픈소스 피드백 포털(Self-hosted)을 발굴하고 서버에 세팅하여 개발 리소스를 획기적으로 단축했습니다.

### 3. 클라우드 인프라 및 원격 개발 환경 구축 (Cloud & DevOps)
*   **클라우드 IDE 도입 및 온보딩 지원:** 주니어 팀원이 로컬 노트북 PC의 성능 저하(CPU/RAM 과부하)로 개발에 어려움을 겪자, Github Codespaces 및 Oracle Cloud(OCI) 기반의 웹 IDE 환경을 세팅해 주어 물리적 장비의 한계를 극복하고 개발 생산성을 즉각적으로 복구했습니다.
*   **비동기 원격 협업 환경 세팅 (글로벌 근무 환경):** 본인의 미국(시애틀) 출국으로 인한 12시간의 시차를 극복하기 위해, Github PR 중심의 비동기 코드 병합 프로세스를 정립하고 일일 화상 회의 체계를 세팅하여 24시간 끊김 없이 개발이 돌아가는 업무 환경을 조성했습니다.
*   **배포 자동화 및 컨테이너 관리:** Kubernetes(K8s, K3s)와 Docker 컨테이너를 활용한 개인 리눅스(Ubuntu) 서버 환경을 구축하고, Github과 연동하여 PR 병합 시 자동으로 변경 사항이 배포(CI/CD)되도록 인프라를 운영했습니다.

### 4. 핵심 비즈니스 로직 설계 및 이해관계자 조율 (PM & Domain Engineering)
*   **소프트웨어 대가 산정 및 인력 매칭 알고리즘 설계:** "공문(RFP)이 들어오면 과거 실적과 자격증 보유 직원을 자동 매칭해 달라"는 경영진의 궁극적인 비즈니스 요구사항을 분석하여, '회사 연혁 - 직원 학력/경력 - 보유 자격증 - 과거 프로젝트 실적 - 업무 분류 코드'가 하나의 세트로 연동되는 데이터 아키텍처를 설계했습니다.
*   **크롤링 및 데이터 동기화 파이프라인 기획:** 한국소프트웨어산업협회의 '소프트웨어 대가 산정 가이드'가 매년 갱신될 때마다 이를 사람이 직접 확인하던 노가다 작업을 없애기 위해, 웹 크롤링 및 구글 스프레드시트 자동 동기화를 통한 데이터 업데이트 파이프라인을 기획했습니다.
*   **요구사항 조율 및 스코프(Scope) 관리:** 임원진(원장님)이 쏟아내는 수많은 아이디어와 마감일 압박 속에서, 기술적 구현 가능성과 개발팀의 리소스 한계를 명확히 설명하며 개발자의 업무 과부하를 막는 방어막(Shield) 역할을 충실히 수행했습니다.

### 5. 기술 리더십 및 주니어 멘토링 (Tech Leadership & Mentorship)
*   **주니어 프론트엔드 개발자 역량 강화:** 신규 합류한 팀원에게 Git/Github를 활용한 버전 관리, 브랜치 전략, 클라우드 환경 세팅법을 직접 화면 공유로 멘토링하며 실무 적응력을 높였습니다.
*   **비개발자 대상 커뮤니케이션 및 시연 롤플레잉:** 코드를 모르는 경영진을 상대로 성과를 발표할 때 기술적인 디테일(Github 커밋 등)보다 직관적인 UI 시연과 비즈니스 가치 위주로 설명하도록 역할극(롤플레잉) 방식의 모의 발표를 주도하며 팀원의 소프트 스킬을 향상시켰습니다.
*   **개발 문화 및 심리적 안정감 조성:** 주니어 팀원이 마감일 압박과 모호한 기획으로 스트레스를 받을 때, "비즈니스 요구사항을 우선순위화하는 것은 리더의 몫"임을 명확히 인지시키고 업무 압박을 덜어주어 팀원이 개발에만 집중할 수 있는 긍정적인 문화를 만들었습니다.
