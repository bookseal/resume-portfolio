# 포트폴리오 — Bithabit (소규모 스터디 그룹용 습관 추적 앱)

- **라이브 서비스**: https://habit.bit-habit.com
- **저장소**: https://github.com/bookseal/bithabit
- **상관 성과**: IITP 해커톤 2등, 2년 이상 무료 운영 중 (출시 2년 이상 경과한 현재까지 핵심 유저 활성 유지)

## 개요
약 2년 전에 개발하여 현재까지 무료로 운영 중인 소규모(3~5인) 스터디용 습관 형성 서비스입니다. 지속적인 핵심 유저들이 매일 참여하며 높은 리텐션을 기록하고 있습니다.

## 문제 의식 및 차별점
기존의 '챌린저스' 같은 습관 형성 앱들은 참여율을 높이기 위해 인증 절차가 간단하지만(예: 계단 사진만 찍기, 물통 사진만 찍기), 이는 실질적인 '증명'이 부족하여 장기적인 동기부여와 습관 형성에는 한계가 있습니다. 다른 블로그 스터디 등에서는 사용자가 수동으로 타임랩스를 찍어 공유하고 방장이 일일이 확인해야 하는 불편함이 있었습니다.

이러한 문제를 해결하기 위해, **별도의 스톱워치 없이 웹캠을 통해 자동으로 타임랩스 비디오(GIF)를 생성해주는 시스템**을 구축했습니다. 캡처 시점이 랜덤한 간격으로 잡히고 날짜·사용자 정보가 오버레이 되어 사진 재사용 등의 조작이 어렵고, 확실한 '집중'을 증명할 수 있어 신뢰도 높은 스터디 문화를 형성할 수 있었습니다.

## 핵심 흐름 및 기술적 성과
1. **타임랩스 GIF 자동 생성**: 사용자가 '공부 시작'을 누르면 일정 간격으로 웹캠 프레임을 캡처하고, '공부 종료' 시 이를 하나의 GIF로 자동 병합합니다.
2. **무거운 영상 연산의 클라이언트 처리 (서버 비용 최소화)**: GIF 병합이라는 무거운 연산을 서버가 아니라 브라우저의 **Web Worker(gif.js)** 에서 클라이언트 단으로 처리하여 서버 연산 부하와 비용을 크게 낮췄습니다. 인증·채팅·세션 기록 등 가벼운 로직만 백엔드가 담당하는 구조라 2년 이상 저비용 무료 운영이 가능했습니다.
3. **실시간 채팅 및 인증 연동**: 생성된 GIF는 **WebSocket** 기반 실시간 채팅방에 바로 공유되며, 백엔드(FastAPI)가 세션·출석 데이터를 기록하여 사용자의 참여 현황(세션 추적·주간 보드·개인 캘린더)을 자동으로 집계·시각화합니다.
4. **무비밀번호 인증 (이메일 OTP)**: 비밀번호 없이 이메일로 OTP를 발송·검증하는 방식과 JWT 기반 세션을 적용하여 가입 마찰을 줄였습니다.

## 기술 스택
- **프론트엔드**: Flutter Web (Dart) — provider 상태관리, gif.js + Web Worker(GIF 생성), `dart:html` 기반 WebSocket 채팅, shared_preferences(토큰 저장)
- **백엔드**: FastAPI (Python) + uvicorn, SQLAlchemy + SQLite, python-jose(JWT), websockets, Gmail SMTP(OTP 발송)
- **인프라/배포**: Docker, k3s(경량 Kubernetes), Traefik(TLS 인그레스), nginx(정적 웹 서빙)

## DevRel 및 제품 운영 관점
해커톤 수상에 그치지 않고, 2년이라는 긴 시간 동안 사용자의 니즈를 파악하며 실제 동작하는 프로덕션 레벨의 라이브 서비스를 유지보수하고 있다는 점에서, 안정적인 시스템 아키텍처 설계와 사용자 중심의 문제 해결 역량을 입증합니다. 최근에는 세션 추적·주간 보드·개인 캘린더·카메라 폴백 모드·다국어(i18n) 등 사용자 경험을 지속적으로 개선하고 있습니다.
