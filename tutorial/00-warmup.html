Warm-up 면접 당일 · 30분 손풀기

작동하는 작은 것 하나 — fetch → map 컴포넌트

큰 튜토리얼을 완주할 시간이 없을 때, 라이브 코딩에서 실제로 나오는 단 하나의 패턴을 손으로 칩니다: 데이터를 불러와(fetch) 목록으로 렌더(map). npm·빌드 없이 이 HTML 하나로 됩니다.

배우는 것

useState · useEffect · fetch · .map() 렌더 · loading / error 세 상태. 프론트 라이브 코딩의 80%가 이 조합.

완료 기준

버튼을 누르면 API에서 사용자 목록을 불러와 카드로 보여준다. 로딩 중·에러도 화면에 나온다.

왜 이거 하나면 되나

체험형 인턴 라이브 코딩은 "React 마스터"를 안 본다. "데이터를 화면에 뿌릴 줄 아는가, 상태를 다룰 줄 아는가"를 본다. 그게 정확히 fetch → map이다. 이걸 손에 익히면 얼지 않는다.

1. 30초 실행 — 먼저 돌아가는 걸 본다

아래 전체를 warmup.html로 저장하고 브라우저로 열면 바로 작동합니다. React를 CDN으로 불러오고 <script type="text/babel">이 JSX를 즉석 변환합니다 — 설치 0.

warmup.html · 전체 (저장 후 더블클릭)
<!doctype html>
<html>
<head><meta charset="utf-8"><title>fetch → map</title></head>
<body>
  <div id="root"></div>

  <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
  <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
  <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

  <script type="text/babel">
    const { useState, useEffect } = React;

    function UserList() {
      const [users, setUsers]     = useState([]);   // 데이터
      const [loading, setLoading] = useState(false); // 로딩 상태
      const [error, setError]     = useState(null);  // 에러 상태

      async function load() {
        setLoading(true);
        setError(null);
        try {
          const res = await fetch("https://jsonplaceholder.typicode.com/users");
          if (!res.ok) throw new Error("HTTP " + res.status);
          const data = await res.json();
          setUsers(data);
        } catch (e) {
          setError(e.message);
        } finally {
          setLoading(false);
        }
      }

      return (
        <div style={{ font: "16px system-ui", maxWidth: 520, margin: "40px auto" }}>
          <button onClick={load}>사용자 불러오기</button>
          {loading && <p>불러오는 중…</p>}
          {error && <p style={{ color: "crimson" }}>에러: {error}</p>}
          {users.map((u) => (
            <div key={u.id} style={{ border: "1px solid #ddd", borderRadius: 8, padding: 12, margin: "8px 0" }}>
              <b>{u.name}</b> — {u.email}
            </div>
          ))}
        </div>
      );
    }

    ReactDOM.createRoot(document.getElementById("root")).render(<UserList />);
  </script>
</body>
</html>

2. 손으로 짓기 — 5줄씩 이해하며 타이핑

돌아가는 걸 봤으면, 이제 빈 파일에 처음부터 직접 쳐 봅니다. 외우지 말고 이 순서를 이해하세요.

머릿속 순서 (라이브 코딩에서 이대로 말하며 짜면 좋다)
1) 상태 3개를 만든다     → data(빈 배열), loading(false), error(null)
2) load 함수를 만든다    → loading=true → fetch → json → setData → 실패하면 setError → 끝에 loading=false
3) 화면을 그린다         → 버튼 + (loading이면 "로딩") + (error면 에러) + data.map(...)
4) key를 잊지 않는다     → map으로 만든 각 요소에 key={고유값}
5) 언제 부를지 정한다    → 버튼 클릭(onClick) 또는 처음 뜰 때(useEffect(fn, []))
면접에서 자주 꼬는 3곳

① key 빠뜨림 — map 렌더엔 항상 key. ② loading/error 무시 — "성공"만 그리면 미완성. 세 상태를 다 그리는 게 프로답다. ③ fetch 에러 처리 — res.ok 체크 없이 .json()하면 조용히 깨진다.

3. 같은 것을 Next.js + TypeScript로

면접이 Next.js/TS 환경이면 문법만 살짝 다르다 — 로직은 위와 똑같다. 타입만 붙이면 된다.

app/users/page.tsx · "use client" 컴포넌트
"use client";
import { useState } from "react";

type User = { id: number; name: string; email: string };

export default function UsersPage() {
  const [users, setUsers]     = useState<User[]>([]);   // ← 제네릭으로 타입
  const [loading, setLoading] = useState(false);
  const [error, setError]     = useState<string | null>(null);

  async function load() {
    setLoading(true); setError(null);
    try {
      const res = await fetch("https://jsonplaceholder.typicode.com/users");
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const data: User[] = await res.json();
      setUsers(data);
    } catch (e) {
      setError(e instanceof Error ? e.message : "unknown");
    } finally {
      setLoading(false);
    }
  }

  return (
    <div>
      <button onClick={load}>사용자 불러오기</button>
      {loading && <p>불러오는 중…</p>}
      {error && <p style={{ color: "crimson" }}>에러: {error}</p>}
      {users.map((u) => (
        <div key={u.id}><b>{u.name}</b> — {u.email}</div>
      ))}
    </div>
  );
}
TS에서 딱 3개만 기억

① useState<User[]>([]) — 배열 타입. ② type User = {...} — 응답 모양을 타입으로. ③ e instanceof Error — catch의 e는 unknown이라 좁혀줘야 .message 접근.

4. 라이브 코딩 대비 — 확장 3연습 (시간 남으면)

기본이 손에 붙었으면, 면접관이 흔히 얹는 요구를 미리 쳐 봅니다. 각 5분.

확장 과제
A. 처음 뜰 때 자동 로드   → useEffect(() => { load(); }, []);  // 버튼 없이
B. 검색 필터 추가         → 입력창 state + users.filter(u => u.name.includes(q)).map(...)
C. 상세 보기             → 카드 클릭 → 선택된 user를 state로 → 아래에 상세 렌더
AI 도구를 쓰는 모습이 곧 강점

이 역할은 "AI로 10x"를 원한다(메일도 AI 도구 준비를 요청). 라이브 코딩에서 Claude Code로 스캐폴딩하고, 왜 그렇게 짰는지 설명하면, 프론트 깊이의 약점이 "AI-native로 빠르게 만드는 사람"이라는 강점으로 뒤집힌다. 손으로 짤 줄 알되, AI로 가속하는 걸 보여줘라.
