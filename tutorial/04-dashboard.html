2. 문서 메타데이터 (fit report 포함)

어떤 문서들이 있는지, 각 .docx 경로와 미리 계산된 fit report를 데이터로 둡니다.

web/src/lib/documents.tstypescript
export type FitReport = { lines: number; budget: number; ok: boolean; fat: string[] };

export type Doc = {
  slug: "resume" | "coverletter";
  title: string;
  mdFile: string;     // public/examples 아래 파일명
  docxFile: string;
  fit: FitReport;     // 배포본은 mock (실제 계산은 #10에서 Python 연결)
};

export const DOCS: Doc[] = [
  {
    slug: "resume",
    title: "Resume — John Doe",
    mdFile: "resume_johndoe.md",
    docxFile: "resume_johndoe.docx",
    fit: { lines: 52, budget: 55, ok: true, fat: [] },
  },
  {
    slug: "coverletter",
    title: "Cover Letter — John Doe",
    mdFile: "coverletter_johndoe.md",
    docxFile: "coverletter_johndoe.docx",
    fit: { lines: 34, budget: 55, ok: true, fat: [] },
  },
];

export const getDoc = (slug: string) => DOCS.find((d) => d.slug === slug);

백틱 경로 주의

href={`/dashboard/${doc.slug}`} — JSX 속성 안에서 JS를 쓰려면 { }, 그 안에서 문자열 조합은 백틱 템플릿 리터럴을 씁니다. "/dashboard/" + doc.slug도 같습니다.

4. 동적 상세 라우트 [doc]

대괄호 폴더 [doc]는 어떤 값이든 받는 경로입니다. /dashboard/resume → params.doc = "resume".

터미널 · web/
mkdir -p "src/app/dashboard/[doc]"
web/src/app/dashboard/[doc]/page.tsxtsx
import fs from "node:fs";
import path from "node:path";
import { notFound } from "next/navigation";
import Markdown from "react-markdown";
import remarkGfm from "remark-gfm";
import { getDoc } from "@/lib/documents";

// Next 15: params 는 Promise → 반드시 await
export default async function DocPage(
  { params }: { params: Promise<{ doc: string }> }
) {
  const { doc } = await params;
  const meta = getDoc(doc);
  if (!meta) notFound();

  // public/examples 아래 md 파일을 서버에서 읽는다
  const mdPath = path.join(process.cwd(), "public", "examples", meta.mdFile);
  const md = fs.readFileSync(mdPath, "utf8");

  return (
    <main className="mx-auto max-w-3xl px-6 py-16">
      <div className="flex items-center justify-between">
        <h1 className="text-2xl font-bold">{meta.title}</h1>
        <a href={`/examples/${meta.docxFile}`} download
           className="rounded-lg bg-teal-600 px-4 py-2 text-sm font-medium text-white
                      hover:bg-teal-700">
          .docx 다운로드
        </a>
      </div>

      {/* fit report */}
      <div className={`mt-4 rounded-lg border p-3 text-sm ${
        meta.fit.ok ? "border-green-500/40 bg-green-500/10"
                    : "border-amber-500/40 bg-amber-500/10"}`}>
        {meta.fit.ok ? "✓ 한 페이지에 맞음" : "⚠ 한 페이지 초과"} —{" "}
        {meta.fit.lines}/{meta.fit.budget}줄
      </div>

      <article className="cookbook mt-8">
        <Markdown remarkPlugins={[remarkGfm]}>{md}</Markdown>
      </article>
    </main>
  );
}
Why notFound()

/dashboard/asdf처럼 없는 문서로 오면 notFound()가 표준 404 페이지를 띄웁니다. 잘못된 입력을 명시적으로 처리하는 습관 — 나중에 실데이터에서 더 중요해집니다.

배포 미리보기

fs.readFileSync(...public/examples...)는 dev에선 완벽히 되지만, standalone 빌드에선 public/이 자동으로 안 딸려갑니다. #8 Dockerfile에서 public/을 복사해 이 문제를 해결합니다. 지금은 dev에서 되면 OK.
