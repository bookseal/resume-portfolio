1. 마크다운 렌더러 설치

터미널 · web/ 안에서
cd web
npm install react-markdown remark-gfm
패키지 2개의 역할

react-markdown: 마크다운 문자열을 React 요소로 변환. remark-gfm: 표·체크박스 등 GitHub 스타일 마크다운(GFM) 지원 플러그인.

2. 콘텐츠를 문자열 모듈로

실제 skill/SKILL.md의 9단계를 요약한 내용입니다. .ts 모듈에 문자열로 두면 빌드에 자동 포함되어 배포에서도 안전합니다.

web/src/content/cookbook.tstypescript
export const COOKBOOK = `
# paste-jd Cookbook

채용지원을 **변환 파이프라인**으로 다룹니다.
공유 재료(경력·프로젝트·글)를 회사별 JD라는 *함수*에 통과시켜,
한 장짜리 맞춤 이력서를 재현 가능하게 만듭니다.

## 파이프라인 9단계

| # | 단계 | 하는 일 |
|---|------|--------|
| 0 | 입력 확인 | JD에서 회사/역할/식별자 도출 |
| 1 | JD 보존 | 원문 JD를 그대로 저장 |
| 2 | 자산 로드 | history·assets·과거 제출물 재사용 |
| 3 | 이력서 작성 | resume 마크다운 초안 |
| 4 | 자소서 작성 | coverletter 마크다운 초안 |
| 5 | 사실 검증 | 이전 회사명 제거·주장한 스택 확인 |
| 6 | docx 빌드 | 템플릿에 채워 넣고 한 페이지 fit 체크 |
| 7 | 핸드오프 | 포트폴리오 링크·한 장 체크리스트 |
| 8 | 제출 후 | drafts→submitted 이동·기록 갱신 |

## 설계 원칙

- **변하는 것은 데이터로**: 문구는 마크다운, 서식은 .docx 템플릿, 스타일 매핑은 템플릿에서 추론.
- **줄(line)이 예산**: 한 페이지 ≈ 55줄. 세 줄짜리 불릿은 한 줄의 3배 비용.
- **바이트를 검증**: 렌더가 아니라 .docx 내부 XML을 직접 확인해 링크 누수를 잡음.
- **사실 우선**: 요약은 JD 필수 역량과 1:1로 매핑. 없는 경험은 쓰지 않음.
`;
문법 주의

백틱(\`)으로 감싼 템플릿 리터럴이라 여러 줄 문자열이 됩니다. 내용 안에 백틱이나 ${}가 들어가면 이스케이프가 필요하니, 지금은 넣지 마세요.

3. /cookbook 라우트 만들기

src/app/ 밑에 cookbook 폴더를 만들고 그 안에 page.tsx를 두면 /cookbook 경로가 생깁니다.

터미널 · web/ 안에서
mkdir -p src/app/cookbook
web/src/app/cookbook/page.tsxtsx
import Markdown from "react-markdown";
import remarkGfm from "remark-gfm";
import { COOKBOOK } from "@/content/cookbook";

export default function CookbookPage() {
  return (
    <main className="mx-auto max-w-3xl px-6 py-16">
      {/* prose 유틸이 없어도 최소 서식을 위해 간단한 래퍼 클래스 사용 */}
      <article className="cookbook">
        <Markdown remarkPlugins={[remarkGfm]}>{COOKBOOK}</Markdown>
      </article>
    </main>
  );
}

마크다운에 최소 서식 입히기

react-markdown은 순수 태그(<h1><table>…)만 뱉습니다. globals.css 맨 아래에 몇 줄 추가해 보기 좋게 만듭니다.

web/src/app/globals.csscss (파일 끝에 추가)
.cookbook h1 { font-size: 2rem; font-weight: 700; margin: 0 0 1rem; }
.cookbook h2 { font-size: 1.3rem; font-weight: 600; margin: 2rem 0 .5rem; }
.cookbook p, .cookbook li { line-height: 1.7; margin: .5rem 0; }
.cookbook ul { list-style: disc; padding-left: 1.4rem; }
.cookbook table { border-collapse: collapse; width: 100%; margin: 1rem 0; }
.cookbook th, .cookbook td { border: 1px solid #ccc; padding: .4rem .7rem; text-align: left; }
.cookbook th { background: rgba(0,0,0,.04); }
Why Server Component 그대로 두나

이 페이지엔 "use client"가 없습니다 = 기본값인 Server Component. 마크다운을 서버에서 HTML로 변환해 보내므로, 브라우저로 내려가는 JS가 줄어 더 빠릅니다. 클릭·상태가 필요한 화면에서만 client로 바꿉니다(#6에서 처음 등장).
