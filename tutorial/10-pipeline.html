Issue #10 Milestone 4 · Iterate (배포 후)

실제 파이프라인 연결 (아웃라인)

배포가 살아있는 상태에서, mock이던 내부 로직을 실제 Python md2docx로 승격합니다. 이 페이지는 방향만 잡는 아웃라인 — 세부 코드는 배포 후 Claude Code와 함께 채웁니다.

왜 아웃라인만?

#1–#9로 “보이는 SaaS + 실제 로그인 + 배포”가 완성됩니다. 실제 docx 생성은 폴리글랏 이미지(Node+Python)라 난이도가 한 단계 높아, 배포 이후 천천히 붙이는 게 학습에 좋습니다.

아웃라인 단계

  1. 파이프라인 API 라우트: web/src/app/api/pipeline/route.ts에서 child_process(execFile)로 python3 ../tools/md2docx_template.py <md> <out> 호출. 입력 md는 요청 본문 또는 소유자 폴더에서, 결과 docx 경로를 반환.
  2. 폴리글랏 Dockerfile: runner 스테이지에 apk add python3 py3-pip + pip install python-docx, 그리고 저장소의 tools/·templates/를 이미지에 복사. (지금 build 컨텍스트가 web/이므로, 컨텍스트를 저장소 루트로 올리거나 tools를 복사해 넣도록 조정.)
  3. fit report를 실제 계산: mock 숫자 대신 Python report_fit 출력(줄 수/초과 불릿)을 파싱해 #4의 배지에 반영.
  4. 실데이터 이관: 본인 이력서/자소서 md·docx를 서버 hostPath로 옮김
    터미널
    scp -r ~/workspace/chan_이력서_자기소개서_포트폴리오/03_output/* \
  bit-habit:/home/ubuntu/workspace/paste-jd-data/owner/
    Deployment에 PASTE_JD_OWNER_DIR=/data/owner env 추가 → #7의 소유자 섹션이 실데이터를 읽음.
  5. JD 붙여넣기 → 생성 UI: 이름처럼 “paste JD” 입력창을 만들어, 붙여넣은 JD로 파이프라인을 돌려 맞춤 산출물을 뷰어에 표시(초기엔 소유자 전용).
Claude Code와 함께
